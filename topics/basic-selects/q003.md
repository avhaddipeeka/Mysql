# Question 003: Create an Employees Table

### Problem Statement
Now that you have a database, you need to store user information. You are tasked with creating a table that holds basic contact details.

**Task:** Create a table named `employees` with the following requirements:
1.  **id**: An integer that uniquely identifies each person (Primary Key).
2.  **first_name**: A text string (max 50 characters).
3.  **last_name**: A text string (max 50 characters).
4.  **email**: A text string (max 100 characters) that must be unique.
5.  **mobile**: A text string (max 15 characters).
6.  **created_at**: The date and time the record was created (default to the current time).

---

### Key Concepts
*   **PRIMARY KEY**: Ensures each row is unique and not null.
*   **VARCHAR(N)**: Stores variable-length strings up to *N* characters.
*   **UNIQUE**: Ensures no two rows have the same value (e.g., no duplicate emails).
*   **DEFAULT**: Automatically fills a column if no value is provided.

---

### Solution (Standard SQL/MySQL)

```sql
CREATE TABLE employees (
    id INT PRIMARY KEY,
    first_name VARCHAR(50),
    last_name VARCHAR(50),
    email VARCHAR(100) UNIQUE,
    mobile VARCHAR(15),
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```
### PostgreSQL Version
In PostgreSQL, we often use SERIAL for an auto-incrementing ID:
```sql
CREATE TABLE employees (
    id SERIAL PRIMARY KEY,
    first_name VARCHAR(50),
    last_name VARCHAR(50),
    email VARCHAR(100) UNIQUE,
    mobile VARCHAR(15),
    created_at TIMESTAMP DEFAULT NOW()
);
```
